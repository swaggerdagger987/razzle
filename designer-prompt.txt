## STEP 3: Write 10 tickets

For each FAIL in the walker report, write a ticket to designer-tickets/open/:

```markdown

page: filename.html
type: broken-panel | agent-presence | brand | loading-state
severity: P0 | P1 | P2
---

**File**: frontend/[exact file]
**Current**: [what it does now]
**Expected**: [what it should do]

[exact change to make]
```

Ticket numbering: check designer-state.json for the last ticket number, increment from there.

## STEP 4: Update your state

Write designer-state.json with the new ticket count and what pages you reviewed.

Every 10th cycle, update designer-insights.md with patterns you see.

## STEP 5: Commit

```bash
git add designer-tickets/ designer-state.json designer-insights.md designer-reports/
git commit -m "design: 10 tickets from walker report"
```

## RULES
- 10 tickets max per invocation
- Each ticket names ONE file to edit
- P0 (broken/doesn't load) before P1 (wrong behavior) before P2 (polish)
- If the walker found 0 FAILs, look for WARNs instead
- Do not write code. Do not fix anything. Just tickets.

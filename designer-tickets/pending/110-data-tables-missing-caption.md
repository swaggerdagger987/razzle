# DES-110: 82+ data tables missing `<caption>` for screen readers

**Priority**: P2 — affects every data panel
**Category**: Screen reader accessibility
**WCAG**: 1.3.1 (Info and Relationships)

## Problem

Only the Lab screener table has a `<caption>`:
```html
<caption class="sr-only">NFL Player Statistics Screener</caption>
```

All other tables — 45 in standalone HTML pages and 37 generated by lab-panels.js — have no caption. Screen readers list tables by their caption text; without one, every table is announced as an unnamed table. Users can't distinguish "Efficiency Rankings" from "Consistency Rankings" from "VORP Table" in a table list.

## Location

**Standalone HTML pages (45 tables):**
advantage.html, airyards.html, auction.html, career.html, career-compare.html, comptable.html, consistency.html, draftclass.html, drops.html, dualthreat.html, efficiency.html, gamelog.html, gamescript.html, garbagetime.html, handcuffs.html, matchups.html, opportunity.html, playoffs.html, records.html (4 tables), redzone.html, regression.html, reportcard.html, schedule.html, scoring.html, seasonpace.html, snapefficiency.html, stacks.html, stocks.html, streaks.html, successrate.html, targetpremium.html, tdregression.html, tradefinder.html, usage.html, vorp.html, waivers.html, weekly.html, weeklyleaders.html, weeklymvp.html, workload.html, yoy.html

**lab-panels.js (37 tables):** all dynamically created tables.

## Fix

Add `<caption class="sr-only">` after every `<table>` opening tag with a descriptive name matching the panel's purpose. Example:
```html
<table class="pa-table">
  <caption class="sr-only">Positional Advantage Rankings</caption>
  <thead>...
```

For lab-panels.js, add to each table creation string:
```js
html += '<table class="vorp-table"><caption class="sr-only">VORP Rankings</caption><thead>...';
```

## Why This Matters

Dynasty power users who use screen readers need to identify which table they're in. With 70+ panels, each containing at least one data table, the lack of captions makes the product nearly unusable for screen reader users. The pattern already exists in lab.html — it just needs to be applied everywhere.

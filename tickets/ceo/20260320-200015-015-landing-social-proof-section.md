## No social proof anywhere on the landing page -- no testimonials, no user count, no community signal

**PRIORITY: P1** | **Type: structural**
**Page**: index.html
**Found by**: Razzle (CEO Review)

The landing page has zero social proof. No testimonials. No user count. No "as seen on" badges. No Reddit mentions. No screenshots of the tool being used in the wild. For a pre-launch product this is understandable, but the landing page should at least SHOW what social proof will look like and use what's available now.

**BEFORE** (what it is now):
- No social proof section of any kind
- No testimonials, no user mentions, no community references

**AFTER** (what it should be):
- Add a "Built for dynasty managers" section after the screener showcase
- For pre-launch (now): show 3-4 fabricated "screenshot in context" mockups -- what a Razzle screenshot looks like when posted in a Reddit thread, a group chat, or a tweet. Frame it as "this is how managers use Razzle" not as fake testimonials.
- Show the r/DynastyFF and fantasy Twitter community as the target: "Built for the managers who live on r/DynastyFF and fantasy Twitter"
- When real testimonials arrive: replace mockups with real quotes + username attribution
- Add a live counter when possible: "X screener sessions this week" (can be pulled from server analytics)
- Consider adding: "Powered by nflverse data" badge -- fantasy data nerds know and trust nflverse

**WHY**: Social proof reduces the "is this legit?" hesitation. The landing page looks polished enough to be real, but it looks lonely. Even pre-launch, showing the CONTEXT in which the product will be used (Reddit posts, group chats, tweets) helps the visitor imagine themselves using it. "Built for r/DynastyFF" is a tribal signal -- dynasty managers see it and think "this is for ME."

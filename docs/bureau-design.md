## Problem Statement

The Bureau of Intelligence is the conversion engine of razzle.lol. It's where free Screener users become paying Pro users. Currently it's a blank connect screen — Sleeper username input, nothing else. The design doc for the agent-first launch identified the Bureau as the "I need this" moment. It needs to deliver that moment immediately after connection.

The core insight: dynasty managers live in 5 tabs. The Bureau consolidates all of them into one league-aware intelligence briefing delivered by the 6 Razzle agents.

## Demand Evidence

From the prior office-hours session: dynasty managers currently duct-tape 5+ tools together. The Bureau solves this by being the only tool that knows your league. The championship probability number is the viral hook — a single number every manager in the league wants to see and argue about.

## Status Quo

The Bureau page currently shows:
- Sleeper username input
- "Connect" button
- ESPN/Yahoo "coming soon" badges
- Footer

After connection: league selector, basic roster display, some pressure map and self-scout CSS exists (6500 lines in league-intel.html) but the full agent-delivered experience is not built.

## Target User & Narrowest Wedge

Dynasty manager who just used the free Screener and wants to see their own league analyzed. The wedge: "Enter your Sleeper username. See your championship odds in 10 seconds."

## Constraints

- Vanilla HTML/JS/CSS. No frameworks.
- All league data from Sleeper API (client-side fetches).
- Player stats from terminal.db via backend API.
- Monte Carlo runs in-browser (Box-Muller, 10K simulations).
- Must work without LLM inference for page loads — agents deliver static analysis computed from data, not live AI responses.
- Razzle design language: sand background, chunky 3px borders, 4px offset shadows, position colors, Luckiest Guy/Space Mono/Caveat fonts.

## Premises

1. The championship probability number is the viral hook — it's what gets screenshotted and sent to group chats.
2. Self-Scout being free is the right call — genuine free value builds trust and word-of-mouth.
3. Rival intel is the conversion trigger — "I can see MY team but not my opponents? That's worth $10/month."
4. The agents are present in the Bureau — each tab is delivered by the relevant agent character.
5. The Bureau works without LLM calls — data-computed analysis, not AI inference per page load.

### Data Architecture

**Sleeper API (client-side):**
- `GET /user/{username}` → user_id
- `GET /user/{user_id}/leagues/nfl/2025` → all leagues
- `GET /league/{league_id}` → settings, scoring, roster positions
- `GET /league/{league_id}/rosters` → all rosters with player IDs
- `GET /league/{league_id}/users` → manager names + avatars
- `GET /league/{league_id}/transactions` → trades, waivers, drops (multi-season via previous_league_id)
- `GET /league/{league_id}/matchups/{week}` → weekly matchup results

**Backend API (terminal.db):**
- `GET /api/players` → player stats for roster analysis
- `POST /api/screener/query` → filtered player data for depth analysis

**Monte Carlo (client-side, in-browser):**
- Player projection: PPG mean + stddev from last 3 seasons, min 6 games
- Simulate 10,000 seasons: for each manager, sample PPG for each starter, sum weekly scores, determine matchup wins, compute playoff probability
- Championship probability = % of simulations where this manager wins the championship
- Runs in a Web Worker to avoid blocking UI thread
- Re-simulates instantly for scenario explorer ("what if I trade Player X?")

**Manager Profiles (computed from Sleeper transactions):**
- Parse all transactions (trades + waivers + drops) for current + previous seasons
- Track per-manager: trade count, position bought/sold, timing (pre/post loss), FAAB spend
- Behavioral tag assigned from trade pattern analysis:
  - PANIC SELLER: >60% of trades happen within 48h of a loss
  - HOARDER: net positive pick accumulation, low trade frequency
  - AGGRESSIVE: top 20% in trade frequency, willing to overpay
  - STEADY: consistent trade patterns, rarely reacts to losses
  - PATIENT: low trade frequency, waits for value

### Agent Integration (no LLM calls)

Each agent's "commentary" is template-driven, not AI-generated:
- Razzle's briefing: fill-in-the-blank from computed data: "you're {rank}. {assessment based on odds/record/roster}. {top rival} has {strength}. your path is {recommendation}."
- Hawkeye's scout report: position grades + age data → template commentary
- Bones' rival intel: behavioral tags + trade data → template commentary
- Atlas' history: pattern matching on historical data → template commentary

This means: zero AI cost per page load. Templates produce Razzle-voice output from structured data. The Situation Room (separate page) is where live LLM inference happens.

## Error & Loading States

**Loading sequence (5-second target starts from "Connect" click):**
1. Button text changes to "connecting..." (Caveat font, animated dots)
2. Sleeper API calls fire (username → user_id → leagues): skeleton cards appear with "pulling your league data..." (Caveat)
3. League selector appears (if multiple leagues): user picks one
4. Roster + stats + transactions fetch (parallel): position grade skeletons pulse with "hawkeye's reviewing the tape..."
5. Monte Carlo runs (Web Worker): odds hero shows spinning Octo animation with "octo's crunching 10,000 simulations..."
6. Full reveal: hero number fades in, tabs populate

**Error states:**
- Invalid username: "couldn't find that username on Sleeper. double-check the spelling." (Caveat, warm tone)
- Sleeper API down: "sleeper's servers are napping. try again in a minute." (with retry button)
- Rate limited: queue silently, retry with exponential backoff (500ms → 1s → 2s → 4s), show "pulling data... this is taking a sec"
- No dynasty leagues: "looks like you're in redraft leagues. razzle is dynasty-first for now. explore the free Screener while we build redraft support."
- Empty roster: "this league hasn't drafted yet. come back after the draft."
- Broken previous_league_id chain: History tab shows "not enough league history for pattern analysis" instead of empty

**Multi-league selector (appears after connection if user has 2+ leagues):**
- Full-width dropdown or card list showing: league name, format (dynasty/keeper/redraft), team count, your record
- Most recently active league pre-selected
- Sorted by last activity date
- Selecting a league reloads the Bureau with that league's data
- League selector persists in a header bar below the tabs for easy switching

## Monte Carlo Specification

**Inputs:**
- Each player: PPG mean + stddev from player_stats table (last 3 seasons, min 6 games played, seasons with fewer excluded)
- Rookies/insufficient data: use positional ADP-based projections as fallback (mean PPG for that ADP range)
- Each manager's starting lineup: best starters by projected PPG per roster slot

**Simulation per iteration (x10,000):**
- For each player: sample a season PPG using Box-Muller normal distribution (mean, stddev)
- For each week remaining: each manager's weekly score = sum of starter PPGs + small random noise (stddev=5)
- Resolve matchups per the league's actual schedule
- After regular season: rank by wins (tiebreak: total points)
- Top N seeds make playoffs (N from league settings, default 6)
- Single elimination bracket: higher seed gets home advantage (none mechanically, just seeding)
- Champion = winner of final

**Output:** championship probability per manager = wins / 10,000

**Performance:** Runs in Web Worker. 12 teams x 15 starters x 10,000 sims = ~1.8M samples. Target: <2 seconds on modern hardware.

## Agent Template Examples

**Razzle (Overview briefing) — 3 template variations:**
1. Contender: "you're {rank}. {odds}% isn't dominant, but {top_rival} has {weakness}. your path: {recommendation}."
2. Fringe: "you're {rank}. the math gives you {odds}% — that's not nothing, but you need {position} help. {rival} is vulnerable at {position}."
3. Rebuilder: "you're {rank}. {odds}% is honest. this isn't your year. the play: sell {aging_starter} to {rival_who_needs_position} and stack picks."

**Hawkeye (Self-Scout) — per position:**
- Strong: "{position} room is locked. {starter} is a top-{tier} option with {key_stat}. depth is solid."
- Weak: "{position} room needs work. {starter} is aging ({age}) and there's no heir. the tape says buy before the draft."
- Mixed: "{position} room has upside but risk. {starter} is ascending but {bench} is thin. one injury and you're in trouble."

**Bones (Rivals) — per behavioral tag:**
- PANIC SELLER: "{manager} sells after losses. they're at {consecutive_losses} straight. one more and {player} hits the trade block."
- HOARDER: "{manager} has {pick_count} picks stockpiled. they won't trade fair value — but they'll overpay in a panic."
- AGGRESSIVE: "{manager} trades {freq} times per year. they overpay for {position}. bring {player} to the table."

**Atlas (History):**
- Pattern: "last time a team at {record} with {roster_profile} won this league, they made {trade_count} trades in weeks {window}."
- No history: "this league doesn't have enough history for pattern analysis yet. atlas needs 3+ seasons of data."

## Behavioral Tag Classification

**Rules (applied in order, first match wins):**
1. PANIC SELLER: >50% of trades occur within 48h of a matchup loss (trades = roster moves excluding IR stashes)
2. AGGRESSIVE: top 20% in trade frequency AND net negative in pick accumulation (spending picks)
3. HOARDER: bottom 20% in trade frequency AND net positive in pick accumulation (hoarding picks)
4. PATIENT: bottom 30% in trade frequency, no panic pattern
5. STEADY: default — none of the above patterns triggered

**Minimum data:** requires 8+ transactions across 1+ seasons. Below threshold: show "not enough data" instead of tag.

## Open Questions

1. How many seasons of transaction history does Sleeper API provide via previous_league_id chaining? This affects manager profile accuracy.
2. Should the Monte Carlo run automatically on page load, or on a "Run Simulation" button click? Auto-run is cleaner UX but adds load time.
3. Should the scenario explorer ("what if I trade X?") be part of the free tier or Pro?
4. What happens for leagues with fewer than 3 seasons of history? Atlas tab shows "not enough history" or degrades gracefully?

- You chose championship probability as the first thing a user sees. Not roster grades, not trade suggestions — a single number that creates an emotional reaction. You're designing for screenshots, not for completeness.
